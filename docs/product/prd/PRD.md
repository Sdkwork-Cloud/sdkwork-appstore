## 1. 产品定位与愿景

### 1.1 一句话定位

SDKWork App Store 是 SDKWork 生态的应用市场：让用户在一个入口发现、安装、更新、管理跨端应用，让开发者一站式发布、运营、增长，让运营者拥有行业级审核与编排能力。

### 1.2 愿景

打造对标 Apple App Store、Google Play、应用宝的下一代企业级应用市场，覆盖 **发现 → 转化 → 安装 → 留存 → 增长** 全链路，在 PC、H5、移动原生、桌面、小程序多端提供一致且行业领先的视觉与交互体验。

### 1.3 设计哲学

| 原则 | 说明 | 行业参照 |
| --- | --- | --- |
| 编辑式发现 | 不只是货架，而是有温度的策展内容 | App Store Today、Google Play 编辑精选 |
| 转化优先 | 详情页 3 秒内传递价值主张 | 三家共同的 hero + 截图 + 评分 |
| 信任闭环 | 隐私标签、内容分级、安全下载缺一不可 | App Store 隐私标签、Play Data safety |
| 多端一致 | 同一应用跨 PC/移动/H5 体验对齐 | 应用宝多端同步、Google Play 跨设备 |
| 运营自主 | 运营者可配置首页、合集、榜单、Banner | 三家均提供运营后台 |
| 智能驱动 | 推荐与搜索基于行为与画像 | App Store 个性化 Today、Play For You |

### 1.4 边界（与相邻域的职责切分）

| 域 | 归属 | 集成方式 |
| --- | --- | --- |
| 登录/会话/组织 | `sdkwork-appbase`（IAM） | 双令牌 + IAM RBAC |
| 已注册应用主数据 | `sdkwork-platform`（应用注册中心） | listing 创建时校验 app_id |
| 评论/评分/收藏 | `sdkwork-comments` | listing 关联 comments_thread_id，汇总写入读模型 |
| 支付/IAP 结算 | `sdkwork-commerce` | entitlement 通过 connector 同步，不持有结算 |
| 二进制/媒体存储 | `sdkwork-drive` | 媒体与制品引用 drive 资源，不发起新存储 |
| 通知投递 | `sdkwork-notifications` | 事件订阅 + 通知 connector |
| 全文搜索索引 | `sdkwork-search` | catalog 事件驱动同步，应用市场不持有索引 |

### 4.2 模块一：发现与浏览（Discovery）

#### 4.2.1 首页编辑流（Home Feed）

对标 App Store Today + Google Play For You + 应用宝推荐流。

| 区块 | 内容 | 数据来源 | 个性化 |
| --- | --- | --- | --- |
| Hero Banner | 编辑大图轮播（活动/合集/精选应用） | `catalog_featured_slot` | 全局/分群 |
| Today 故事卡 | 编辑文章卡 + 关联应用/合集 | `catalog_collection`（editorial 类型） | 个性化排序 |
| 编辑精选合集 | 横向滚动合集卡 | `catalog_collection` | 个性化 |
| 个性化推荐 | 「为你推荐」应用卡网格 | `listing_metric_snapshot` + 行为画像 | 强个性化 |
| 榜单速览 | 免费/付费/新品 Top 10 入口 | `catalog_chart_snapshot` | 全局 |
| 分类入口 | 圆形分类图标导航 | `category` | 全局 |
| 福利/活动 | 限时活动卡 | `catalog_collection`（event 类型） | 全局 |
| 最近更新 | 已安装应用的新版本提示 | `library_item` + `release` | 强个性化 |

#### 4.2.2 分类目录（Categories）

- 两级分类体系（主分类 + 子分类），对标 App Store 25+ 主分类 + Google Play 题材维度。
- 每个分类页：精选 Banner + 排行榜 + 编辑合集 + 全部应用网格。
- 分类本地化（`category_localization`）。
- 分类图标与配色由运营配置。

#### 4.2.3 榜单与排行（Charts）

| 榜单类型 | 维度 | 数据来源 |
| --- | --- | --- |
| 总榜 | 免费/付费/畅销 | `catalog_chart_snapshot` |
| 分类榜 | 各分类下的免费/付费/新品 | 同上 + category join |
| 飙升榜 | 周环比增长 Top N | `listing_metric_snapshot` 增量 |
| 新品榜 | 上线 30 天内 Top N | `listing` published_at |
| 编辑榜 | 编辑挑选 Top N | `catalog_collection`（chart 类型） |

榜单快照由 `analytics-worker` 定时生成（小时级 + 日级），前端只读快照以保性能。

#### 4.2.4 合集与专题（Collections）

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| editorial | 编辑策划合集 | 「提升效率的 10 款工具」 |
| chart | 榜单合集 | 「本周免费 Top 20」 |
| event | 活动合集 | 「双 11 应用狂欢」 |
| theme | 主题合集 | 「暗黑模式应用精选」 |
| personalized | 个性化合集（运行时生成） | 「为你推荐」 |

每个合集含本地化标题/描述/封面 + 应用列表 + 排序规则。

#### 4.2.5 搜索与发现（Search）

| 子能力 | 说明 |
| --- | --- |
| 关键词搜索 | 支持应用名、开发者、描述、标签检索 |
| 搜索建议 | 输入即联想，基于热词与应用名 |
| 热搜词 | 实时热词榜 |
| 搜索历史 | 用户级历史记录 |
| 搜索筛选 | 分类、价格、评分、年龄分级、更新时间 |
| 搜索排序 | 相关度、热度、评分、最新 |
| 智能联想 | 拼写纠错 + 同义词扩展（由 search 域提供） |
| 空结果推荐 | 无结果时推荐相关应用 |

搜索由 `sdkwork-search` 提供索引，应用市场通过 catalog 事件同步索引。

#### 4.2.6 福利与活动（Welfare & Events）

对标应用宝「福利中心」。

| 形态 | 说明 |
| --- | --- |
| 活动合集 | 限时专题页，含主视觉 + 应用列表 + 倒计时 |
| 优惠标记 | 应用卡标记「限时免费」「打折」 |
| 任务系统 | 预留：下载任务、签到任务（结算由 commerce 域） |
| 兑换码 | 预留：兑换码发放应用权益 |

### 4.3 模块二：应用详情（Listing Detail）

完整对标三家应用详情页，覆盖转化关键要素。

#### 4.3.1 详情页区块（自上而下）

| 区块 | 内容 | 对标 |
| --- | --- | --- |
| ① 应用头部 | 图标 + 名称 + 副标题 + 开发者 + 价格/获取按钮 | 三家共同 |
| ② 评分速览 | 评分 + 评分人数 + 下载量 + 年龄分级 | 三家共同 |
| ③ 截图/视频 | 横向滚动截图 + 视频预览 + 设备框 | 三家共同 |
| ④ 描述 | 折叠展开 + 富文本 + 关键特性 | 三家共同 |
| ⑤ 更新日志 | What's New + 版本历史 | 三家共同 |
| ⑥ 评分与评论 | 大评分 + 分布柱 + 评论流 + 筛选 | 三家共同 |
| ⑦ 隐私与合规 | 隐私标签 + 数据安全 + 权限披露 | App Store 隐私标签 + Play Data safety |
| ⑧ 信息表 | 开发者/分类/版本/大小/兼容性/语言/年龄/更新时间 | 三家共同 |
| ⑨ 应用内购买 | IAP 项预览（名称 + 价格区间） | App Store + Play |
| ⑩ 相似应用 | 关联推荐网格 | 三家共同 |
| ⑪ 开发者其他应用 | 同开发者应用横向滚动 | 三家共同 |
| ⑫ 编辑点评 | 编辑评语（可选） | App Store 编辑点评 |
| ⑬ 支持 | 官网/支持/隐私政策/举报 | 三家共同 |

#### 4.3.2 详情页交互细节

- **截图浏览**：横向滚动 + 点击放大 Lightbox + 视频自动播放（静音）+ 设备框包裹。
- **获取按钮**：状态机 `未安装 → 安装中（进度环）→ 已安装（打开）`；付费应用 `价格 → 购买 → 打开`。
- **收藏**：心形图标，状态切换带微动效。
- **分享**：生成短链 + 复制 + 系统分享。
- **滚动锚点**：侧边/顶部 sticky 导航跳转到各区块。
- **评论筛选**：按评分、版本、设备、时间筛选；可排序（最有用/最新/最高/最低）。
- **版本历史**：可展开查看历史版本更新日志。

### 4.4 模块三：开发者中心（Publisher Console）

对标 App Store Connect + Google Play Console + 应用宝开发者中心。

#### 4.4.1 开发者入驻（Onboarding）

| 步骤 | 内容 |
| --- | --- |
| 1. 注册开发者 | 提交主体信息（个人/企业）+ 联系方式 |
| 2. 实名认证 | 上传资质 + 平台审核（`publisher_verification`） |
| 3. 签署协议 | 开发者协议 + 平台规范 |
| 4. 创建应用 | 绑定已注册应用（`platform` 域 app_id）+ 主分类 |
| 5. 完善资料 | 多语言元信息 + 图标 + 截图 + 视频预览 |
| 6. 提交审核 | 进入审核队列 |

#### 4.4.2 应用管理（App Management）

| 子能力 | 说明 |
| --- | --- |
| 应用列表 | 我的所有应用 + 状态过滤 |
| 基础信息 | 名称/副标题/描述/关键词/分类/标签 |
| 多语言本地化 | 按语言编辑元信息（`listing_localization`） |
| 图标与媒体 | 图标/截图/视频预览/宣传图（Drive 引用） |
| 区域可用性 | 按区域上架/下架（`regional_availability`） |

## 5. 核心用户场景

### 5.1 终端用户：发现并安装应用

1. 用户打开应用市场首页，看到个性化推荐的 Hero Banner 与 Today 故事卡。
2. 滑动浏览编辑精选合集，点击「提升效率的 10 款工具」进入合集页。
3. 在合集页看到应用网格，点击某应用进入详情页。
4. 详情页查看截图、描述、评分、隐私标签，确认价值后点击「获取」。
5. 系统创建下载凭证，客户端下载并安装，安装完成进入「我的应用」。
6. 用户在「我的应用」查看已安装应用，开启自动更新。

### 5.2 终端用户：搜索应用

1. 用户点击搜索栏，看到搜索历史与热搜词。
2. 输入「笔记」，实时看到搜索建议。
3. 提交搜索，看到结果列表（相关度排序）。
4. 使用筛选器（分类=效率、评分≥4、免费）过滤结果。
5. 点击应用进入详情页，查看后安装。

### 5.3 开发者：发布新应用

1. 开发者完成实名认证，进入 Publisher Console。
2. 点击「创建应用」，绑定在 platform 域已注册的 app_id。
3. 填写多语言元信息（名称/描述/关键词）+ 上传图标/截图/视频预览（Drive）。
4. 选择主分类与标签，配置区域可用性。
5. 填写隐私标签与权限披露，完成内容分级问卷。
6. 创建首个版本，上传制品（Drive），填写更新日志。
7. 选择发布渠道（正式/内测），配置灰度策略。
8. 提交审核，进入审核队列。
9. 审核通过后，应用自动发布到正式渠道。

### 5.4 开发者：版本迭代

1. 开发者在 Publisher Console 选择已有应用。
2. 创建新版本，上传新制品，填写更新日志。
3. 配置灰度（10% → 50% → 100%），提交审核。
4. 审核通过后，按灰度策略推送。
5. 在数据分析页查看安装/留存/崩溃指标。
6. 若发现问题，可暂停灰度或回滚到上一版本。

### 5.5 运营者：审核与编排

1. 运营者登录 Backend Admin，进入审核中心。
2. 查看待审核队列，分配任务给审核员。
3. 审核员查看应用详情 + 制品 + 合规信息，做出决议。
4. 决议记录（通过/拒绝 + 原因 + 政策引用）通知开发者。
5. 运营者切换到运营编排，配置首页 Banner 与编辑合集。
6. 配置榜单规则，调度 analytics-worker 生成榜单快照。
7. 在数据看板查看大盘与运营效果。

### 5.6 CI 客户端：自动检查更新

1. 应用运行时调用 open-api `appstore.releases.checkUpdate`。
2. 系统匹配应用 + 渠道 + 灰度，返回最新可用版本。
3. 客户端调用 `appstore.artifacts.resolveDownload` 获取签名 URL。
4. 客户端下载并校验制品，完成更新。

## 10. 非功能性需求

| 维度 | 要求 |
| --- | --- |

| 国际化 | 多语言（zh-CN/en-US/ja-JP/ko-KR 等）；多区域；多币种预留 |
| 无障碍 | WCAG 2.1 AA；键盘可达；屏幕阅读器 |

## 11. 风险与依赖

| 风险/依赖 | 影响 | 缓解 |
| --- | --- | --- |
| search 域未就绪 | 搜索能力降级 | Phase 1 用 DB like 兜底，Phase 2 切换 search 域 |
| comments 域未就绪 | 评分评论缺失 | Phase 1 用占位，Phase 2 集成 |
| commerce 域已删 | 付费/IAP 缺失 | 预留 entitlement connector，付费结算延后 |
| drive 配额 | 媒体/制品存储受限 | 与 drive 域协商配额 + 冷热分层 |
| 平台多端壳能力差异 | 体验不一致 | Phase 3 各端原生壳，统一设计系统 |
